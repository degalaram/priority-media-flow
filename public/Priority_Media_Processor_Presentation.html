Priority Media Processor

Backend System for Asynchronous Job Processing

A robust system for managing media processing jobs with priority-based queues, lifecycle management, and resource tracking.

1 / 8

Project Goal

Build a robust backend system that manages asynchronous media processing jobs with:

  • Job Submission - Accept processing requests via REST API
  • Priority Queues - Route jobs to dedicated high/low priority queues
  • Background Processing - Handle intensive computation with Celery workers
  • Lifecycle Tracking - Track job states from submission to completion
  • Resource Monitoring - Log CPU and memory usage for each job
  • Status Reporting - Provide real-time job status via API
2 / 8

Technology Stack

Django

Python web framework for building the REST API endpoints and handling job submissions.

Celery

Distributed task queue for background job processing with multi-queue support.

Redis

In-memory message broker for fast communication between API and workers.

PostgreSQL

Relational database for persistent storage of jobs and metrics data.

Frontend Demo

React + TypeScript + Tailwind CSS + Vite for the demonstration interface.

How It Works

  • Step 1: Client sends job request to Django REST API
  • Step 2: API validates input and saves job to database
  • Step 3: Job is routed to Redis based on priority
  • Step 4: Celery worker picks up and processes the job
  • Step 5: Results and metrics are saved to PostgreSQL

API Endpoints

POST /api/v1/submit_job

Submit a new media processing job

  • Input: Source file, target format, priority level
  • Action: Validates, saves to database, routes to queue
  • Output: Job ID for tracking

GET /api/v1/job/{id}/status

Retrieve job status and resource metrics

  • Input: Job ID
  • Output: Current status, all timestamps, metrics (if completed)
